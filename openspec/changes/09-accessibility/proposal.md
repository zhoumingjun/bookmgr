# 09-accessibility: 前端特教适配

## 背景

特殊教育场景下的绘本阅读应用面向两类核心用户群体：特教教师和家长。教师的日常工作需要在设备上长时间浏览、筛选和审核绘本内容；家长的使用场景则多为在家庭环境中陪伴孩子进行阅读。调研表明，这两类用户在视力条件、操作习惯和使用目的上存在显著差异——教师普遍需要更大的字体、更高的对比度和更清晰的功能入口，而家长则更需要简洁的阅读引导和孩子的阅读历史追踪。

现有系统在界面设计上采用统一的 Ant Design 默认主题，未能针对这些差异化需求提供支持。当前系统的字号固定为 14-16px，配色遵循标准企业级设计规范，对视觉条件不佳的用户（尤其是低视力特教教师和老年家长）存在可访问性不足的问题。此外，系统未根据用户角色（teacher / parent / admin）提供差异化的功能入口和页面布局，导致用户体验与角色需求不匹配。

特教适配是本项目的核心产品价值，而非附加功能。本次提案将从前端可访问性改造入手，构建可扩展的适配层，为后续更深入的特教场景优化奠定基础。

## 目标

本次提案旨在实现以下四个核心目标：

**目标 1：4 级字体缩放体系**
建立从 16px 到 32px 的 4 级字体缩放方案（正常 1x / 大 1.25x / 特大 1.5x / 超特大 2x），覆盖全站所有文本内容及 PDF/EPUB 阅读器内部文字，并确保缩放设置在 localStorage 中持久化，用户下次访问时自动恢复。

**目标 2：高对比度配色方案**
在默认 Ant Design 配色的基础上增加可切换的高对比模式，提供纯白背景+纯黑文字的高对比配色，并支持移除装饰性图片的纯文本模式。所有配色切换通过 CSS 变量系统实现，无需重新加载页面。

**目标 3：教师端与家长端差异化界面**
根据登录用户的 role 字段（teacher / parent / admin）自动路由至对应的功能入口。教师端（/teacher/*）包含绘本浏览、我的收藏、提交审核、我的反馈四个功能模块；家长端（/parent/*）包含绘本浏览、我的收藏、我的反馈和阅读记录四个功能模块。两个端点在侧边栏、功能按钮和页面布局上保持一致性，但在可用功能上存在明确区分。

**目标 4：触摸友好与焦点可访问性**
将触摸目标尺寸从默认的 32px 提升至 44px（符合 WCAG 2.1 AAA 标准），调整间距和内边距以适配缩放系数，并为所有交互元素提供清晰的焦点指示（3px solid #0066cc outline）。

## MVP 范围

MVP（Minimum Viable Product）阶段聚焦于最基础的可访问性改造，确保核心用户（特教教师和视力受限的家长）能够正常使用系统。

### MVP 功能列表

1. **字体缩放**：4 级缩放（1x/1.25x/1.5x/2x），通过 CSS rem 单位 + CSS 变量实现，PDF.js 阅读器内通过 CSS transform 同步缩放，设置持久化至 localStorage。
2. **高对比模式**：单开关切换高对比配色（纯白背景 #FFFFFF + 纯黑文字 #000000），通过 body 添加 `.high-contrast` class 实现，关闭后完全恢复默认 Ant Design 配色。
3. **角色路由**：登录后根据 JWT payload 中的 role 字段自动跳转至 /teacher/* 或 /parent/* 路由。admin 保留原有 /admin/* 路由。
4. **触摸目标尺寸**：在字体放大模式下自动将按钮和交互元素的 min-height 从 32px 提升至 44px。
5. **焦点可见性**：在字体放大和高对比模式下统一应用 3px solid #0066cc 的 focus outline。

### MVP 排除项

以下功能不进入 MVP，将在后续批次中实现：
- 纯文本模式（移除装饰性图片）
- 阅读进度同步（跨设备）
- 屏幕阅读器（NVDA/VoiceOver）的完整 ARIA 审计与优化
- 减少动画模式（reducedMotion）
- 家长端的孩子账户管理

## 影响范围

本次改造 **仅限于前端代码**，对后端 API 和数据库无任何影响。具体影响范围如下：

| 模块 | 影响说明 |
|------|----------|
| `frontend/src/` | 新增 AccessibilityContext、角色路由中间件、CSS 变量系统 |
| `frontend/src/pages/teacher/` | 新增教师端页面（复用现有 book/user 组件） |
| `frontend/src/pages/parent/` | 新增家长端页面（复用现有 book/user 组件） |
| `frontend/src/components/` | 新增 AccessibilitySettings 组件（字号/对比度切换开关） |
| `frontend/src/auth/` | 修改 AuthContext，增加基于 role 的路由跳转逻辑 |
| `frontend/src/index.css` | 新增 CSS 变量定义和缩放规则 |
| `frontend/public/index.html` | 无需修改 |
| `backend/` | **无改动** |
| 数据库 / API | **无改动** |

## 验收标准

### 字体缩放

- [ ] 在任意页面点击「字号 → 超特大（2x）」后，全站字体从 16px 变为 32px（CSS 变量生效）
- [ ] 刷新页面后字号设置保持为「超特大」，无需重新选择
- [ ] 在 PDF 阅读器中以 2x 缩放打开绘本，PDF 内文字同步放大
- [ ] 切换字号时页面不闪烁、无布局抖动

### 高对比模式

- [ ] 开启高对比模式后，页面背景变为 #FFFFFF，正文文字变为 #000000
- [ ] 主要按钮（提交、登录等）颜色变为高饱和色（#0066CC 蓝色），对比度 ≥ 7:1
- [ ] 关闭高对比模式后，页面完全恢复 Ant Design 默认配色
- [ ] 高对比模式设置同样持久化至 localStorage

### 角色路由

- [ ] 以 role=teacher 登录，访问 / 跳转到 /teacher/browse
- [ ] 以 role=parent 登录，访问 / 跳转到 /parent/browse
- [ ] 以 role=admin 登录，访问 / 跳转到 /admin/dashboard
- [ ] 未登录用户访问 /teacher/* 或 /parent/* 自动跳转至 /login
- [ ] 切换浏览器标签页不丢失角色状态

### 教师端功能

- [ ] 侧边栏显示：绘本浏览、我的收藏、提交审核、我的反馈
- [ ] 绘本详情页显示「提交审核」按钮，点击后弹出审核提交表单
- [ ] 反馈表单支持选择反馈类型（绘本内容、界面问题、功能建议）

### 家长端功能

- [ ] 侧边栏显示：绘本浏览、我的收藏、阅读记录、我的反馈
- [ ] 绘本详情页不显示「提交审核」入口
- [ ] 阅读记录页面按时间倒序列出已读绘本，显示阅读进度百分比
- [ ] 反馈表单仅支持「功能建议」和「其他」两类

### 触摸友好

- [ ] 在 2x 缩放模式下，按钮的 min-height 从 32px 自动增大至 44px
- [ ] focus outline 在所有交互元素上清晰可见（3px solid #0066cc）
- [ ] 表单输入框在放大模式下保持足够的内边距（padding ≥ 12px）

## 技术风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| CSS 变量覆盖冲突 | Ant Design 组件有大量内联样式，CSS 变量可能无法完全覆盖 | 通过 ConfigProvider 注入 Ant Design 的 token 系统，将变量映射到 design token |
| PDF.js 字体缩放同步 | PDF.js 渲染在 canvas 上，CSS 字体缩放可能不生效 | 使用 CSS transform: scale() 包裹 PDF viewer DOM，配合 viewport 缩放参数 |
| 路由跳转与状态同步 | JWT payload 在 localStorage 中，刷新后需重新解析 role | 在 App 根组件的 useEffect 中解析 token，确保每次挂载时 role 有效 |
| localStorage 存储上限 | 偏好设置简单对象，不超过 1KB，风险极低 | 仅存储 { fontScale: 1.25, highContrast: false }，不会超限 |
